Quem é a W3C?

A W3C, que significa World Wide Web Consortium, é um grupo que cuida da Web (a internet como a gente conhece). Ela foi criada em 1994 por Tim Berners-Lee, o mesmo cientista que inventou a Web.

A W3C existe para criar regras e padrões que ajudem a manter a internet funcionando bem para diferentes navegadores e dispositivos. No começo da internet, cada navegador (como o Chrome, Firefox, etc.) podia funcionar de um jeito diferente, o que causava confusão e erros. Além de atualizações contantes como o HTML5 que traz um HTML mais estrutural para acessibilidade e semântico para mecanismo de buscas. Assim promovendo inovação, democracia e acessibilidade, como o WAI - Web Acessibility Initiative.

As mudanças do HTML5

(Voltar)

Sabemos que o HTML recorrentemente vem tido melhorias, e o foco principal dessas mudanças é em um HTML estruturado, deixando a questão de estilização para o CSS. Com isso, várias tags novas foram criadas com o intuito de um html semântico tanto para os crawlers quanto para os leitores de tela, implementando também a acessibilidade nesta parte, além do WAI com as técnicas WCAG.

Algumas dessas tags, invés da utilização frequente de div's foram substituídas por estruturas, como main, header, footer, nav, section, aside, article, etc. Todas essas tags e entre outras, já dizem quais conteúdos elas abordam, deixando claro para quem está visualizando o conteúdo, visivelmente para nós humanos e para robôs.

WAI - Padrões da W3C para promover acessibilidade

(Voltar)

A WAI (Web Accessibility Initiative) é uma iniciativa da W3C criada para garantir que a Web seja acessível a todas as pessoas, inclusive aquelas com deficiência — visual, auditiva, motora, cognitiva, entre outras.

E para isso a WAI conta com WCAG que vem para estabelecer técnicas para tornar o ambiente tech mais inclusivo. As WCAG (Web Content Accessibility Guidelines) são um conjunto de diretrizes criadas pela WAI que ensinam como tornar o conteúdo de um site acessível para todos — inclusive para quem usa tecnologias assistivas como leitores de tela, navega apenas com o teclado ou precisa de maior contraste de cores.

Exemplos práticos de boas práticas WCAG:

  • Usar alt nas imagens;
  • Garantir contraste suficiente entre texto e fundo;
  • Fazer menus navegáveis pelo teclado;
  • Evitar usar apenas cor para transmitir uma informação;
  • Fornecer legendas e transcrições para conteúdos em áudio/vídeo;
  • Escrever links com descrições claras (ex: “Saiba mais sobre o curso” em vez de “Clique aqui”).
